# 使用者操作手冊

本手冊說明如何在 Google 試算表編輯資料，並將成果發佈到網站。操作不需要程式基礎。

## 一、系統概要

網站的資料來自兩本各自獨立的 Google 試算表：

| 試算表 | 負責內容 | 對應選單 | 產出檔案 |
|--------|----------|----------|----------|
| 圖譜試算表 | 節點、連線（知識圖譜本體） | 圖譜工具 | graph.json |
| 圖片試算表 | 每個節點對應的照片 | 圖片同步 | images.json |

兩本試算表互相獨立，各自有自己的選單與按鈕，請分開操作。

發佈流程如下：

```
在試算表編輯　→　按選單的推送按鈕　→　資料上傳 GitHub　→　等待數分鐘　→　網站更新
```

請注意，按下推送並顯示成功後，網站不會立即更新，需要等待約五分鐘。這是系統的快取機制造成的，屬於正常現象，詳見第六節。

## 二、圖譜試算表 Graph Sheet（節點與連線）

### 分頁命名規則

系統依分頁名稱的開頭判斷用途。名稱大小寫不限，但開頭關鍵字必須正確。

| 分頁名稱開頭 | 用途 | 欄位順序 |
|--------------|------|----------|
| nodes_ | 節點資料，可有多個分頁（例如 nodes_Main、nodes_人物） | A=ID、B=Group、C=Info、D=Start Year、E=End Year、F=Lon、G=Lat |
| link_ | 連線資料，可有多個分頁 | A=Source、B=Target、C=Group、D=Label、E=Info |
| nodes_color | 節點顏色設定 | A=Group、B=Color（十六進位色碼） |
| links_color | 連線顏色設定 | A=Group、B=Color（十六進位色碼） |

### 欄位填寫說明

節點分頁（nodes_）：

- A 欄 ID：節點的唯一代號。此代號同時是圖片試算表配圖的依據，請固定使用，不要任意更改。
- B 欄 Group：分類，決定節點顏色，需在 nodes_color 有對應設定。
- C 欄 Info：說明文字。同一個 ID 出現在多個分頁時，Info 會自動合併。
- D、E 欄年份：可留空。留空表示永久存在，不會被時間軸過濾。
- F、G 欄經緯度：供未來地圖功能使用，可留空。

連線分頁（link_）：

- A 欄 Source、B 欄 Target：填入節點 ID，且這兩個 ID 必須已在某個 nodes_ 分頁定義過，否則會被標記為缺漏。
- C 欄 Group：連線分類，決定線的顏色，需對應 links_color。
- D 欄 Label、E 欄 Info：連線的文字說明，可留空。

### 自動檢查與顏色標記

按下檢查或推送後，系統會自動在試算表上色提示：

| 顏色 | 意義 | 處理方式 |
|------|------|----------|
| 淺紅（節點分頁 B 欄） | Group 定義衝突，同一個 ID 在不同 nodes_ 分頁被設為不同 Group | 統一該 ID 的 Group |
| 淺黃（連線分頁 A、B 欄） | 節點缺漏，連線用到的 ID 沒有任何 nodes_ 分頁定義 | 到 missing_node 分頁查看清單，補進 nodes_ |
| 無色 | 資料正常 | 不需處理 |

missing_node 分頁會自動生成，列出所有缺漏節點與來源位置。補齊後重新執行即會清空。

### 操作步驟

1. 編輯 nodes_ 或 link_ 分頁。
2. 點上方選單「圖譜工具」，選擇：
   - 「檢查節點 / 更新圖譜標記（不推送）」：只檢查並上色，不發佈。建議先用此項確認沒有紅色與黃色標記。
   - 「推送 graph.json 到 GitHub」：檢查並發佈。
3. 查看跳出視窗的結果，訊息對照見第五節。

## 三、圖片試算表 Media Sheet（節點配圖）

### 運作方式

工具以檔名前綴的方式，把雲端硬碟的圖片配給節點：

- 在 image_ 分頁的每一列填入一個前綴。
- 工具會遞迴掃描指定的雲端硬碟資料夾，包含所有子資料夾。
- 檔名以該前綴開頭的圖片，全部歸入該列（一個前綴可對應多張圖）。

例如前綴為 A01，硬碟中有 A01_1.jpeg、A01_2.jpeg、A01_封面.png，則三張都會收入。

### 分頁與欄位

只處理名稱開頭為 image_ 的分頁，欄位順序如下：

| 欄 | 名稱 | 用途 |
|----|------|------|
| A | node_id | 寫入 JSON 的節點 ID，必須與圖譜的節點 ID 完全一致，網站才會把圖掛到該節點 |
| B | file_name | 當作前綴使用，工具以此欄去硬碟比對檔名開頭 |
| C | info | 圖片說明，可留空 |
| D | website_sync_info | 輸出參考欄，不影響比對 |

### 填寫注意事項

兩個一致性是配圖成敗的關鍵，務必確認：

1. A 欄 node_id 必須與圖譜試算表的節點 ID 完全一樣。
2. B 欄前綴必須與雲端硬碟的圖檔名稱開頭完全一樣。

任一處相差一個字，包含空格、繁簡、異體字或大小寫，就會配不到，導致網站該節點點開後沒有圖。

特別注意容易寫錯的字。例如人名「陳溪州」與「陳溪洲」，州與洲是不同字。本專案已統一為「陳溪洲」，圖譜節點 ID、圖片 node_id、圖檔名稱前綴三處都要用「陳溪洲」，不可混用。新增資料時請同樣留意這類同音異字。

其他注意：
- 圖檔權限需設為「知道連結的任何人可檢視」，否則網站會顯示破圖（Image restricted）。
- 不要在不同子資料夾放同名圖片，否則會被列為重複檔名，img_id 會互相覆蓋。

### missing_img 分頁

每次同步後自動生成，列出三類問題：

| 問題描述 | 意義 | 處理方式 |
|----------|------|----------|
| 找不到圖檔 | 此前綴在硬碟找不到任何圖 | 檢查前綴拼字，或上傳、改名圖檔 |
| 重複檔名 | 不同資料夾有同名圖，會互相覆蓋 | 將其中一張改名 |
| 孤兒圖檔 | 硬碟有此圖，但沒有任何前綴配到 | 新增一列前綴對應，或忽略（不會上網站） |

### 操作步驟

1. 整理雲端硬碟圖檔，確認檔名前綴正確、權限已開。
2. 編輯 image_ 分頁，填好 A 欄 node_id 與 B 欄前綴。
3. 點上方選單「圖片同步」，選擇「推送 images.json 到 GitHub」。
4. 查看跳出結果，並檢查 missing_img 分頁。

## 四、進階除錯：在網頁點節點查看 Console

在網站上點任一節點，瀏覽器主控台（按 F12，切到 Console）會印出該節點對應到幾張圖與檔名，例如：

```
[MediaSection] node "A01" → 3 png  ["A01_1.jpeg", "A01_2.jpeg", "A01_封面.png"]
```

若數字為 0 但預期有圖，表示圖片試算表的 node_id 沒有對上圖譜的 ID，請檢查空格、繁簡與大小寫。

## 五、推送結果說明

按下推送後，跳出視窗可能顯示下列訊息：

| 訊息 | 意義 | 處理方式 |
|------|------|----------|
| 推送成功 HTTP 200 或 201 | 已上傳 GitHub | 等待數分鐘後查看網站 |
| 資料無變動，跳過推送 | 內容與上次相同 | 正常，不需處理 |
| JSON 為空，停止推送（圖片） | 沒有配到任何圖 | 檢查前綴與圖檔 |
| 找不到 GITHUB_TOKEN | 金鑰未設定 | 聯絡管理者，參見 TOKEN_設定說明.md |
| HTTP 401 | 金鑰失效或權限不足 | 聯絡管理者重新產生 token |
| HTTP 404 | repo 路徑錯誤 | 聯絡管理者 |
| 其他失敗訊息 | 不明錯誤 | 截圖訊息提供管理者 |

## 六、發佈與等待

### 為什麼按了成功網站還是舊的

資料上傳 GitHub 後，網站是透過 GitHub 的 raw 快取讀取資料。這層快取大約每五分鐘才更新一次。在這段時間內，無論如何重整，網站都仍是舊資料，屬於正常情況。

### 正確的等待與確認方式

1. 確認推送視窗顯示「推送成功」。
2. 等待約五分鐘，有時需要五到十分鐘。
3. 在瀏覽器按 Ctrl + Shift + R 強制重整。
4. 若仍是舊的，再等一輪即可，不要重複按推送。

### 改節點或圖片內容需要請工程師重新發佈網站嗎

不需要。只要是修改試算表內容（節點、連線、配圖），按推送按鈕並等待五分鐘即可，網站程式碼不必更動。只有要修改網站的外觀或功能時，才需要工程師重新部署，相關說明見開發技術文件。

## 七、常見問題

選單不見了：重新整理試算表頁面（按 F5），「圖譜工具」或「圖片同步」會重新出現。

按了沒反應：查看畫面右下角的提示與跳出視窗。第一次使用會要求授權，請依畫面同意。

某張圖沒出現：多半是前綴與 node_id 對不上，或圖片成為孤兒。對照 missing_img 修正。

網站破圖（顯示 Image restricted）：該圖檔在雲端硬碟的權限需設為「知道連結的任何人可檢視」。

等很久還是舊的：直接開啟下列網址，查看 GitHub 上的資料是否已更新，並以此為準。

```
https://github.com/ms-112-scott/Zhudong/blob/master/src/data/graph.json
```

若此處已是新資料、網站仍是舊的，表示還在等待快取，再等一段時間即可。若此處也是舊的，表示推送沒有成功，請重新推送。

其他錯誤：截圖訊息提供管理者或工程師。
