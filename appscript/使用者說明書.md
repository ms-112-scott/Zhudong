# 使用者說明書

在試算表編輯節點，按一個按鈕把圖片資料同步到網站。


## 運作方式

工具讀每個 nodes_ 分頁的節點 id，拿 id 當前綴去雲端硬碟找圖。
圖檔名開頭是 `{id}_` 的全部算這個節點的圖（一個節點可多張）。

例：節點 id = `A01`，雲端硬碟有 `A01_1.jpeg`、`A01_2.jpeg`、`A01_封面.png` → 三張都收。


## 操作

1. 編輯 nodes_ 分頁、整理雲端硬碟圖檔
2. 上方選單 同步工具 → 推送 images.json 到 GitHub
3. 看跳出結果：
   - 推送成功：完成，網站幾分鐘後更新
   - 資料無變動：跟上次一樣，正常
   - JSON 為空：沒配對到任何圖，檢查 id 與圖檔前綴
   - 同步失敗：截圖訊息給管理者


## 命名規則

- 節點 id 放 nodes_ 分頁的 A 欄（跟圖譜用的同一個 id）
- 圖檔名要 `{id}_` 開頭，底線後面隨意（`A01_main.jpg`、`A01_2.png` 都行）
- id 跟前綴要完全一致，差一個字（空格、繁簡、大小寫）就配不到
- 圖放指定的雲端硬碟資料夾，子資料夾也會掃


## missing_img 分頁

同步後自動產生。列出「孤兒圖檔」= 雲端硬碟裡有、但沒有任何節點 id 配對到的圖。

| 欄位 | 意思 |
|------|------|
| 圖片名稱 | 檔名 |
| 圖片位置 | 在雲端硬碟的資料夾路徑 |
| 問題描述 | 為何被列出 |

處理方式：要嘛把圖改成正確前綴，要嘛在 nodes_ 補上對應節點。改完重新同步，清掉的會從清單消失。


## 常見問題

按了沒反應：看跳出視窗。沒有選單就按 F5 重整頁面。

網站圖沒更新：先確認是「推送成功」，推送後等幾分鐘，再 Ctrl+F5 強制重整瀏覽器。

某張圖沒出現：多半圖檔前綴跟節點 id 對不上，或圖跑進清單變孤兒。對照 missing_img 修。

圖在雲端硬碟看得到但網站破圖：該圖檔權限要設「知道連結的任何人可檢視」。

其他錯誤：截圖給管理者。
